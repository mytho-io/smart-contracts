# üìú Project Structure  

## TotemFactory.sol  
üí° **Purpose:** Creates totems and stores totem data.  

### Functions:  
- `createTotem(metaData, name, symbol)` ‚Äì Creates a new totem with a new `TotemToken`.  
- `createTotemWithExistingToken(uint256 tokenId)` ‚Äì Creates a new totem with an existing ERC20/ERC721 token. Merit system for the created totem activates immediately.  
- `addTokenToWhitelist(address tokenAddr)` ‚Äì Adds an existing token to the whitelist.  
- `removeTokenFromWhitelist(address tokenAddr)` ‚Äì Removes a token from the whitelist.  

---

## Totem.sol *(Governance Contract)*  
üí° **Purpose:** Governance logic for each totem. Uses `ProxyBeacon` as part of OpenZeppelin's `UpgradeableBeacon` system. All Totem contracts share a common implementation.  

### Functions:  
- `meritBoost()` ‚Äì Earn merit for the totem holder during the Mythum subperiod.  
- `collectMYTH()` ‚Äì Collect accumulated `MYTH` from `MeritManager`.  

---

## TotemDistributor.sol *(Totem Token Sale & Distribution)*  
üí° **Purpose:** Handles `TotemToken` distribution, sales, and burning. Uses an oracle for `MYTH/USD` conversion.  

### Functions:  
- `buy(uint256 amount)` ‚Äì Buy `TotemTokens` during the sale period.  
  - **Limits:**  
    - Maximum **5,000,000** tokens per address.  
    - **Price:** $0.00004 per `TotemToken`.  
- `sell(uint256 amount)` ‚Äì Sell `TotemTokens` during the sale period.  

### Conditions:  
- When all tokens are sold:  
  - Merit system **activates**.  
  - `Buy/Sell` **becomes unavailable**.  
  - `burnTotems()` **becomes available**.  
  - `TotemToken` **becomes transferable**.  

### MYTH Distribution:  
- **2.5%** ‚Üí `revenuePool`.  
- **0.5%** ‚Üí Totem creator.  
- **Remaining** ‚Üí Totem's treasury.  
- **Send liquidity to AMM.**  
- **Received LP sent to Totem‚Äôs treasury.**  

### Additional Functions:  
- `burnTotems()` ‚Äì Burn `TotemTokens` and receive `MYTH` tokens in return.  
  - `MYTH` share is proportional to the user's `TotemToken` share in circulation.  
- `exchangeTotems()` ‚Äì Exchange custom tokens for `MYTH` from the Totem‚Äôs treasury.  
  - Custom tokens are sent to the Totem‚Äôs treasury.  

---

## MeritManager.sol *(Merit System Controller)*  
üí° **Purpose:** Manages merit accumulation and distribution. Tracks Mytho periods.  

### Functions:  
- `boostTotem(address totemAddress, uint256 amount)` ‚Äì Called once per period by a `Totem` contract to increase merit balance.  
- `collectMYTH()` ‚Äì Claim accumulated `MYTH` for a `Totem` contract.  
- `creditMerit(address totemAddress, uint256 amount)` ‚Äì Credit merit manually to a selected `Totem` based on off-chain actions.  
- `addToBlacklist(address totemAddress)` ‚Äì Add totem to blacklist.  
- `removeFromBlacklist(address totemAddress)` ‚Äì Remove totem from blacklist.  

---

## MYTHVesting.sol *(MYTH Distribution Vesting Contract)*  
üí° **Purpose:** Handles `MYTH` distribution via vesting.  

---

## TotemToken.sol *(ERC20 Token with OP Compatibility)*  
üí° **Purpose:** Custom ERC20 token, non-transferable until the end of the sale period.  

### Functions:  
- `constructor()` ‚Äì Mints **1,000,000,000** tokens, distributed as follows:  
  - **250,000** ‚Üí Totem creator.  
  - **100,000,000** ‚Üí Totem treasury.  
  - **899,750,000** ‚Üí `TotemDistributor`.  
- `transfer()` ‚Äì Disabled during the sale period.  

---
